# Contributing Guidelines

Thank you for considering contributing to this repository.

## Contribution Rules

- Follow clean code practices
- Maintain existing project structure
- Write meaningful commit messages
- Do not break existing functionality

## Testing

- Test changes locally before committing
- Ensure no dependency conflicts

## Folder Discipline

- Do not mix project files
- Keep reports, notebooks, and screenshots organized

## Suggestions

- Raise an issue before major changes
- Use pull requests for improvements

This repository follows professional internship and industry standards.
